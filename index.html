Projects

Peppol for the masses!

Project lead: Triantafullenia Doumani

In this project, we extend the popular "Peppol" e-invoicing network. In Peppol Classic (current Peppol) a four-corner model is used wherein each participant needs to connect to Peppol via a licenced gateway. This has some advantages such as KYC assurance, but also some disadvantages (gateways can be expensive to use, they can read and even edit your business-sensitive invoice data, and the KYC system only works for legal entities from a specific list of countries).

We propose "Peppol Classic + Peppol Direct = Peppol Hybrid", adding an optional feature of sending an invoice straight to its recipient, identified by a WebID URL.

See our peppol-python and peppol-python-demo repos for more info.

Sciencemesh for Nextcloud

Project lead: Michiel de Jong

Technology partner: {muze}

In this project, we extend the innovative "Sciencemesh" project to Nextcloud. Sciencemesh builds on Open Cloud Mesh (OCM) to enable federation between enterprise sync and share systems. It is spearheaded by the three-year EU-funded project "CS3MESH4EOSC", to whom we as Ponder Source are proud to be a sub-contractor.

Collaborative Invoice Composition

Project lead: Michiel de Jong

Technology partner: m-ld

The vision behind this project (which is still in a very early stage) is that a buyer and a seller can collaborate in real-time to determine the contents of an invoice. See this GitHub issue for more details.
